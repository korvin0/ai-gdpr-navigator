Фаза 2 : Интерактивный список мер с кнопками «Сделано/Инфо».
Логика Фазы 2 — это превращение «сырых» данных из Google Sheets в интерактивный рабочий инструмент. После того как в Фазе 0 и Фазе 1 мы собрали все флаги (Дети, Скрейпинг, GenAI, GDPR Status), бот должен сформировать персональную дорожную карту.
Вот подробная бизнес-логика для разработчика:

1. Алгоритм фильтрации (Backend-логика)
Бот не просто показывает таблицу, он выполняет SQL-подобный запрос к вкладке Content_Checklist.
Правила отбора пунктов:
    1. Базовый уровень: Выбираются все строки, где Trigger = "always".
    2. Юридический статус (из Фазы 1):
        ◦ Если GDPR_Status = "Mandatory", выбираются пункты из Листов 1–9.
        ◦ Если GDPR_Status = "Anonymous", Листы 1–9 скрываются (кроме общих), остаются только меры безопасности.
    3. Специфические триггеры (из Фазы 0):
        ◦ Если is_child = True ➡️ добавить пункты с тегом is_child.
        ◦ Если is_gen_ai = True ➡️ добавить пункты с тегом is_gen_ai.
        ◦ Если has_scraping = True ➡️ добавить пункты с тегом has_scraping.
        ◦ Если is_high_risk = True ➡️ добавить пункты с тегом is_high_risk.

2. Структура меню (Frontend в Telegram)
Для удобства пользователя бот группирует отфильтрованные пункты по «Блокам».
Главное меню Фазы 2: Бот присылает сообщение: «Ваш чек-лист готов! Он разделен на тематические блоки. Пройдите по каждому, чтобы завершить аудит».
Кнопки (Inline):
    • 🟦 Блок А: Право и Базис (Листы 1-3)
    • 🟩 Блок Б: Данные и Сбор (Листы 4-6)
    • 🟧 Блок В: Права и Прозрачность (Листы 7-9)
    • 🟥 Блок Г: Безопасность и AI Act (Лист 10)
    • 📊 Посмотреть прогресс (Показывает, сколько % пройдено)

3. Интерактивная работа с пунктом (Внутри блока)
Когда пользователь нажимает на «Блок Б», бот присылает первый пункт из этого блока.
Вид сообщения:
Пункт 5.2: Соблюдение robots.txt При веб-скрейпинге необходимо технически исключить сбор данных с ресурсов, которые запрещают это через robots.txt или ToS.
[ ✅ Сделано ] [ ℹ️ Подробнее ] [ ➡️ Пропустить ]
Логика кнопок:
    • ✅ Сделано: Бот помечает этот пункт в памяти сессии как выполненный и присылает следующий пункт.
    • ℹ️ Подробнее: Бот показывает текст из колонки Hint (из Google Sheets). Поскольку мы работаем без Gemini в этой фазе, это статичная подсказка из вашей таблицы.
    • ➡️ Пропустить: Бот просто переходит к следующему вопросу, оставляя этот в статусе «To Do».

4. Динамические подсказки (Замена Gemini)
Вместо ИИ мы используем «всплывающие» подсказки или вторые сообщения.
    • Если пользователь нажал «ℹ️ Подробнее», бот берет данные из колонки Detailed_Instruction.
    • Пример контента из таблицы: "CNIL рекомендует использовать библиотеки вроде Scrapy с настроенным ROBOTSTXT_OBEY = True. Это докажет вашу добросовестность при аудите".

5. Обработка прогресса
Разработчик должен хранить временный список ID пунктов, которые пользователь отметил как «Сделано».
    • В любой момент пользователь может нажать «Прогресс».
    • Бот выдает: «Выполнено 12 из 28 обязательных мер. Осталось разобрать: Блок В, Блок Г».

Резюме для разработчика (что запрограммировать):
    1. Функция FilterChecklist: Принимает профиль проекта ➡️ возвращает массив ID из Google Sheets.
    2. Pagination (Пагинация): Механизм показа пунктов по одному (чтобы не спамить).
    3. Session Cache: Хранение ответов пользователя (Done/Todo) до конца сессии.
    4. UI: Inline-кнопки для навигации между блоками и управления конкретным пунктом.
Фаза 2 завершается, когда все блоки «просмотрены». Переходим к Фазе 3: Генерация финального отчета и подключение Gemini AI для ответов на оставшиеся вопросы?
