Фаза 0: Профилирование (Настройка фильтров)
Это «входной шлюз» бота. Задача этой фазы — за 4–5 быстрых кликов определить технический стек и целевую аудиторию проекта. Ответы пользователя активируют соответствующие переменные (флаги), которые потом будут использоваться для фильтрации контента из Google Sheets.

1. Список вопросов и логика переменных
Бот поочередно присылает сообщения с Inline-кнопками «Да» / «Нет».

Сообщение бота

Что активирует в таблице

Ваш ИИ является генеративным? (создает текст, изображения, код или аудио)
is_gen_ai
Пункты Листа 10: водяные знаки, защита от галлюцинаций, формат safetensors.

Пункты Листа 4 и 7: усиленная минимизация, упрощенная инфо-графика для детей.

Требования AI Act: обязательный DPIA, человеческий надзор и логирование.

2. Технический алгоритм для разработчика (FSM)
Разработчик должен использовать Finite State Machine (FSM). Состояния (States) переключаются после каждого ответа:
    1. State START: Пользователь нажал /start. Бот кратко объясняет цель («Проверим ваш ИИ на соответствие нормам CNIL и GDPR»).
    2. State GET_GENAI: Присылает Вопрос 1 ➡️ Записывает ответ в user_data['is_gen_ai'] ➡️ Переход к State GET_CHILD.
    3. State GET_CHILD: Присылает Вопрос 2 ➡️ Записывает ответ в user_data['is_child'] ➡️ Переход к State GET_SCRAPING.
    4. State GET_SCRAPING: Присылает Вопрос 3 ➡️ Записывает ответ в user_data['has_scraping'] ➡️ Переход к State GET_RISK.
    5. State GET_RISK: Присылает Вопрос 4 ➡️ Записывает ответ в user_data['is_high_risk'].

3. Завершение фазы и мостик к Фазе 1
После того как последний флаг (is_high_risk) получен, бот делает краткое резюме профиля, чтобы пользователь видел, что его поняли правильно.
Пример сообщения:
«Профиль настроен! ⚙️
    • Проект: Генеративный ИИ
    • Аудитория: Взрослые
    • Источник: Скрейпинг
Теперь перейдем к самому важному — определим правовой статус вашей модели относительно GDPR.»
